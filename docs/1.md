### First step :

"**A scan request comes in, gets validated, checked against policy, turned into explicit workflow state**"

* no actual scan yet

<br>

```
app/
├── config.py
├── storage/
│   └── schemas.py
├── scanner/
│   └── profiles.py
├── utils/
│   └── validation.py
└── agent/
    └── state.py
```

**+ `.env.example` :** template for the required `.env` :

```
APP_NAME=Nolly
APP_ENV=dev
DEBUG=true
ALLOW_PRIVATE_TARGETS=true
DEFAULT_PROFILE=baseline
```

* Later local Docker or cloud runners can use different settings

<br>

**`app/config.py` :** basic app configuration management

**`app/scanner/profiles.py` :** for fixed scan profiles, bounded runtime, allowed options

* defines what kind of scans N0lly is allowed to run

**`app/storage/schemas.py` :** implement the record including state condition

**`app/agent/state.py` :** help system track and explain what happens during a scan

**`app/utils/validation.py` :** "`policy layer`" target validation and scope enforcement should happen outside the model (LLM should not decide whether a target is valid)

<br>

##### Tools :

**`validate_target_tool.py` :** actually validate the target host

**`check_scope_tool.py` :** use whitelist to validate the scope
