#### 1. Adding thin agent layer on top of our current cycle

**Later it :**

* reads current `ScanRecord`

* reasons for next steps

* calls only the approved tools

> #### What is Strands ?
> 
> **Agent = reasoning loop + tools**
> 
> * **can only call tools**, not running raw commands, bypass allowlist
> 
> `without agent` :
> 
> ```python
> if service == "http":
>     do_web_followup()
> ```
> 
> `with agent` :
> 
> ```
> Agent:
> "I see HTTP service → maybe web follow-up?"
> 
> → calls decide_followup_tool
> → system enforces rules
> ```
> 
> Flow :
> 
> 1. read state
> 
> 2. think
> 
> 3. choose tool
> 
> 4. call tool
> 
> 5. observe result
> 
> 6. repeat

<br>

**`agent/prompts.py` :** defines the identity and behavioral rules of the future agent

* agent's `operating instructions`

* **"if we later use LLM, what instructions should it always follow"**

**`agent/tools_registry.py` :** registry of all tools the agent is allowed to use

* a controlled toolbox, manage the tool permissions for the agent

**`agent/nolly_agent.py` :** the agent reasoning layer

* `analyze_record()` : we read the record to make a reasoning
  
  * interpreting the scan results

**`modified main.py` :** new endpoint `/scans/{scan_id}/agent-analysis`

* for the agent's interpretation of the scan, simply look up scan record then send record to `agent.analyze_record(record)` then return the agent's output
