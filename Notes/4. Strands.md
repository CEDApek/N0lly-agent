> * create agent

### Integrating + new endpoint `POST /scans/{scans_id}/agent-run`

> within the new endpoint, Strands agent can :
> 
> * loot at a scan
> 
> * decide whether to wait, process, or summarize
> 
> * call only the approved tools

**`prompts.py` :** this is the system prompt for the agent, behavior guidance

* tells the model :
  
  - you are Nolly
  - you cannot execute arbitrary commands
  - you must only use provided tools
  - you must respect scope and profiles
  - if a scan is still running, do not pretend it is finished
  - if a report exists, summarize it instead of reprocessing

$\rightarrow$ must ensure model not hallucinate findings, not recommending actions outside policy, etc.

**`models.py` :** for Structured output based on agent's decision similar to `Pydantic`

```python
class AgentDecision(BaseModel):
    action: Literal["wait", "process_scan", "summarize_report", "inspect_scan"]
    rationale: str = Field(description="Short explanation for why this action was chosen")
    user_message: str = Field(description="User-facing explanation of the current scan state")
    recommended_followup_profile: Optional[str] = Field(
        default=None,
        description="Approved follow-up profile if relevant",
    )
```

**`tools_strands.py` :** tool wrappers, giving the agent a tools via `@tool`

**Current given tools :**

* `get_scan_report`

* `process_scan_record`

* `get_scan_job_status`

* `get_scan_snapshot`

**`nolly_agent.py` :** This is the real Strands agent class

* Instantiate `Agent`, pass on the `SYSTEM_PROMPT`, pass tools, call it with a prompt

* Structured output via `AgentDecision`

> **This is important, as it is a bridge** between :
> 
> * model backend
> 
> * prompt
> 
> * tool

**Added FastAPI endpoint :** to call the Strands agent through API

**What it does :**

- load scan record
- if missing → 404
- run the Strands agent on that scan
- return the structured decision

**Why this is useful**

Now you can call one endpoint and let the agent decide:

- wait
- process_scan
- summarize_report
- inspect_scan

```
User / client
  -> FastAPI endpoint /agent-run
     -> NollyStrandsAgent.run_for_scan(scan_id)
        -> model reads prompt
        -> model calls tools if needed
           -> get_scan_snapshot
           -> get_scan_job_status
           -> maybe process_scan_record
           -> maybe get_scan_report
        -> model returns AgentDecision
     -> FastAPI returns structured result
```
