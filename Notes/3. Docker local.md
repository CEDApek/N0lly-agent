## Build Docker lab + First Local Runner

> ##### What Agentic AI do
> 
> 1. calls `submit_scan_tool`
> 
> 2. `submit_scan_tool` calls `runner_local.py`
> 
> 3. `runner_local.py` uses fixed profile templates
> 
> 4. execution happens within guardrail
> 
> **Cloud side :** Docker (isolated, service composition)
> 
> **AI red teaming :** safe behavior, handles output correctly

<br>

**`lab/docker-compose.yml` :** creates the controlled target lab

* to scan my own local docker network

```yaml
target-web:
  image: nginx:alpine
  container_name: target-web
  networks:
    - nolly-lab
```

* Starts a lightweight Nginx web server container named `target-web`

and other target such as `target-multi` :

```yaml
target-multi:
  image: python:3.11-alpine
  container_name: target-multi
  command: >
    sh -c "
    python -m http.server 80 &
    python -m http.server 8080 &
    while true; do sleep 3600; done
    "
  networks:
    - nolly-lab
```

* starts a python-based container with two HTTP servers (multiple ports opened)

* creates a private Docker network called `nolly-lab`

<br>

**`/scanner/runner_local.py` :** this is the real execution runner for scans

* Contains which Nmap commands belong to for each profile

* `run_local_scan()` : the function that actually runs the scan :
  
  * takes `target`, `profile`, then validate first and assemble the command then run Nmap + result

* Command is stored to `cmd = PROFILE_COMMANDS[profile] + [target]`
  
  * executed via `subprocess.run()` to run the command

**`storage/artifacts.py` :** artifacts storage config

* create folder for each job + write files for the job + read files back later

* after some files produced after scanning $\rightarrow$ it saves them $\rightarrow$ job_store remembers the paths $\rightarrow$ later can be read

**Improved `job_store` :** more record objects (+ artifacts + return code + etc.)

  1. Creates a job
  
  2. runs real Nmap scans (still within host)
  
  3. Stores results as artifacts
