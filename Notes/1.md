#### **Main.py :** `app/main.py`

* ```
  Client
     ↓
  POST /scans
     ↓
  validate_target_tool
     ↓
  check_scope_tool
     ↓
  create ScanRecord
     ↓
  log decisions
     ↓
  set workflow step
     ↓
  save record
     ↓
  return scan record
  ```

* We created the `/scans` endpoint :
  
  ```python
  @app.post("/scans", response_model=ScanRecord)
  def create_scan(request: ScanRequest) -> ScanRecord:
      target_check = validate_target_tool(request.target)
      if not target_check["ok"]:
          raise HTTPException(status_code=400, detail=target_check)
  
      scope_check = check_scope_tool(
          target=target_check["normalized_target"],
          requested_profile=request.requested_profile,
          authorization_ref=request.authorization_ref,
      )
      if not scope_check["ok"]:
          raise HTTPException(status_code=403, detail=scope_check)
  
      record = ScanRecord(
          target=target_check["normalized_target"],
          requested_profile=request.requested_profile,
          approved_profile=scope_check["approved_profile"],
          authorization_ref=request.authorization_ref,
          scope_status=scope_check["scope_status"],
          followup_allowed=False,
          metadata={
              "validation_reason": target_check["reason"],
              "scope_reason": scope_check["reason"],
          },
      )
  
      add_decision(record, f"Target validated: {target_check['normalized_target']}")
      add_decision(record, f"Scope approved with profile: {scope_check['approved_profile']}")
      set_step(record, "scope_approved")
  
      save_scan_record(record)
      return record
  ```
  
  * which validate the target first, if validation fails, returns `400 Bad Request`
  
  * Then do the scope check
  
  * Then create a `ScanRecord` object storing :
    
    ```
    target
    requested_profile
    approved_profile
    authorization_ref
    scope_status
    followup_allowed
    metadata
    ```
    
    * from `schemas.py` : request format & response format already made
  
  * Then we add `add_decision` from `state.py` which adds entries to the decision history
  
  * Then update workflow step, with the states :
    
    ```
    scope_approved
    scan_submitted
    scanning
    parsing_results
    report_generated
    ```
  
  * And save a record in storage (currently in-memory)

#### Create scan job, track job state, query job status

> Previously API only stopped at target valid and approved, we now create job which has a life cycle, and we can track its status

**`app/scanner/job_store.py` :** simulates what later becomes :

* Docker runner

* Kubernetes Job tracking

* database-backed job states

**`app/tools/submit_scan_tool.py` :** implements safe agentic pattern :

* accepts approved target, approved profile

* Then calls `job_store`'s `create_job` to create the actual job

* later job's `xml` and `stdout.log` will will be put to `artifacts`

**`get_job_status_tool.py` :** to retrieve the current state of a job (from job_id)

**update `app/main.py` :** modified `POST /scans` so it submits a job <u>after scope approval</u>

<br>

**Current flow :**

```
API request
  ↓
validate_target_tool (validate the scanning target)
  ↓
check_scope_tool (Checking the target scope)
  ↓
submit_scan_tool (create job after validating)
  ↓
job created in job_store
  ↓
job status updated
  ↓
artifacts attached
  ↓
get_job_status_tool can inspect job
```

* lifecycle : **queued, running, done**

>     }
> ```
